# Global instructions

## Communication

- Reply in Russian
- Code, comments, commits — in English
- No filler text, no repeating the question in the answer

## Code style

- Minimal readable code, follow the project's existing style
- Do not add docstrings, type hints, or comments unless asked
- Do not refactor code that was not mentioned in the request
- No over-engineering
- Use the linter/formatter the project already has configured

## Testing

- Run the project's test suite before suggesting a commit
- Do not modify reference/fixture data unless explicitly asked

## Git

- Short imperative lowercase commit messages
- Do not commit or push without explicit request
- No --force or --amend without confirmation

## Safety

- Never commit .env files, secrets, or tokens
- Never hardcode credentials
